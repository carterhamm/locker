# Locker — Monetization & Distribution

## Pricing Model

### Freemium
- **Free:** Up to 3 services stored, 100 key fetches/month
- **Pro ($9/month):** Unlimited services, unlimited fetches, usage logs, priority support
- **Team ($29/month):** Shared key vaults, team member access, admin controls

Freemium is critical — developers won't pay before they trust you with their keys.

## Distribution Strategy

### Launch Channels (Day 7)
1. **Hacker News — Show HN** — highest-leverage single post for developer tools
2. **r/ClaudeAI** — direct audience of Claude Code users
3. **r/LocalLLaMA** — AI developer community
4. **X/Twitter** — tag @AnthropicAI, @OpenAI, developer influencers
5. **npm listing** — good README = organic discovery

### README Is Your Landing Page
Developers find CLI tools via npm and GitHub. Your README needs:
- One-line description
- 30-second install + usage example
- Security explanation (they will ask "why should I trust you with my keys")
- Link to dashboard

### Long-Term
- GitHub README badges ("Secured by Locker")
- Integration guides for Claude Code, Cursor, Codex
- Blog posts on dev.to and Hashnode
- Approach services (Resend, Stripe, etc.) for Model B partnerships once users exist

## The Trust Problem

**This is the #1 adoption blocker.** Developers are paranoid about third parties handling their API keys. Address it head-on:

- Open source the CLI and encryption layer
- Publish a security whitepaper
- "We never see your keys in plaintext" messaging everywhere
- Offer self-hosted option eventually
